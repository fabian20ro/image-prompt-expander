ERNIE prompt format

For ordinary images, build every expanded prompt in this recommended order:
1. Subject: exact identity, quantity, appearance, pose, expression, and action.
2. Description and details: setting, time, weather, materials, props, foreground/background, atmosphere, and explicit spatial relationships.
3. Style and medium: one clear primary medium or visual tradition, with at most one compatible secondary influence.
4. Technical and capture finish: framing, viewpoint, subject placement, depth, camera/lens or rendering method when relevant, lighting, palette, focus, texture, and finish.

For posters, infographics, interfaces, documents, and comics, lead with the overall format and layout before describing content. State grid, panel count, reading order, placement, typography, and every visible text string explicitly. Use these category patterns when applicable:
- Portrait: subject, clothing/accessories, expression, lighting, background, camera/lens, finish.
- Product: product and surface, lighting setup, palette, props, commercial style, mood.
- Poster/infographic: purpose, exact layout, exact text/content, style, palette.
- Comic: panel count and layout, each panel in reading order, exact dialogue, overall style and coloring.
- Landscape/concept art: overall composition, scene layers, scale, atmosphere, style, lighting.

Prompt requirements

- Preserve every explicit user constraint verbatim. Vary only unspecified details.
- Aim for 50–150 words per expanded prompt.
- Describe what must be visible. Never emit vague placeholders such as "some text", "a button", "relevant facts", or "and so on".
- Preserve user-supplied visible text exactly and wrap each visible string in single quotes. Do not translate it.
- Use concrete positions: left/right, foreground/background, above/below, inside/behind, evenly spaced, panel number, and reading order.
- Avoid contradictory media, styles, lighting, camera, palette, layout, or weather instructions.
- ERNIE has no negative-prompt input. When exclusions matter, state them concretely in the prompt, for example "a clean desk with no visible cables".
- Keep the style description consistent across a series unless the user explicitly asks to explore styles.
- Use UTF-8. English, Chinese, and mixed-language visible text are allowed.
- Do not write inference steps, guidance scale, quantization, or numeric output resolution into the prompt; the application controls them.

Tracery grammar requirements

- Use #rule# references, never square-bracket placeholders.
- Use at most 8 rules, including "origin".
- Keep locked requirements in "origin" or another single-option rule.
- Every rule intended to vary must contain exactly 7 distinct, coherent alternatives. Five or six are acceptable only when seven genuinely distinct choices would reduce quality. Never use 2–4 alternatives.
- Include at least one varying rule.
- Make every alternative grammatically compatible in every expansion.
- Do not duplicate or lightly rephrase alternatives merely to reach the count.
- Output strict JSON with double-quoted JSON strings and no trailing commas.
